Discuss the scope of var, let, const

var ,let, const all are ways of defining variables.but there are different's in them.let's talk about that.

var : before es6- js developer used to use var to define a variable.but it got some limitations.also problem with scope.like on same scope if you use var you can define var with same name many time which is known as reassign also you can reassign value of that same variable.it can be access able from its parent scope

let & const : On es6 there are 2 new ways to define a variable which is let & const.You can't reassign const or let on a scope .but you reassign value let but not const .like on same scope if you use const or let you can't define let or const with same name many time.it's not access able from a different scope

Tell us the use cases of null and undefined

Null and undefined may look like same but actually they aren't.There are different between the two.

undefined is something that we get when we define a variable without and value or we didn't return any value of a function but put that as a value of a variable. on the other hand null is close to undefined but this time we put null as a value to act like there isn't any value.
